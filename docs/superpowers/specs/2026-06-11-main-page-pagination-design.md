# 메인 페이지 배경/그리드/페이징 개선 설계

날짜: 2026-06-11 / 상태: 승인됨

## 배경 (문제)

1. **배경 끊김**: `main.html`의 `html,body{height:100%}` 때문에 body 높이가 뷰포트로 고정되어,
   스크롤 영역에서 배경 그라데이션이 반복되며 경계선이 보인다.
2. **상품 1열 나열**: `.grid`에 `display:grid`와 열 정의가 빠져 있어(미디어쿼리에만 존재)
   데스크톱에서 상품 카드가 세로 1열로 나열된다.
3. **페이징 UX**: `이전 / "1 / 2" / 다음` 텍스트만 있고 숫자 클릭 이동이 불가능하다.

## 결정 사항

1. **배경**: `html{height:100%}` + `body{min-height:100%}`로 변경, `background-repeat:no-repeat` 추가.
2. **그리드**: 데스크톱 기본 2열(`repeat(2, minmax(0,1fr))`), 기존 미디어쿼리(≤980px 2열, ≤640px 1열) 유지. — 사용자 선택: 2열
3. **페이지 크기**: `MainController`의 `@PageableDefault` size 12 → **10**.
4. **숫자 네비게이션**:
   - 컨트롤러가 0-based `beginPage`/`endPage` 윈도우(최대 5개, 현재 페이지 중심)를 계산해 모델에 추가.
   - 템플릿: `이전 | 1 … 4 [5] 6 … 12 | 다음`. 현재 페이지 active 강조, keyword 유지.
   - 윈도우 시작 > 0이면 맨 앞 `1` 링크, 시작 > 1이면 `…` 추가. 끝도 대칭.
   - 총 페이지 ≤ 5면 ellipsis 없이 전부 표시. 다크 모드 스타일 포함.

## 테스트

- `MainControllerMvcTest`: ① 기본 size=10, ② 중간 페이지 윈도우(12페이지 중 0-based 4 → begin=2, end=6),
  ③ 마지막 페이지 경계(0-based 11 → begin=7, end=11). 뷰 렌더링은 슬라이스 테스트가 main.html을
  실제 렌더링하므로 템플릿 문법 오류도 함께 검증된다.
